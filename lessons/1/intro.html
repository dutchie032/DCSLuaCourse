A little context on Lua in DCS

Lua was designed as a lightweight, efficient and embeddable language.
It is used a lot in the gaming world, however it is not only used here. There's medical products I know use it as well to customise parts of the core products for instance for specific countries, vendors etcetera.
For DCS Specifically it can be an extension of the mission editor, since when making missions the Mission Editor can sometimes have it's limits.
Very big multiplayer, event driven or special missions can be incredibly hard to accomplish through only the flag system.
On top of that maintaining it can be even a lot worse. One change and you need to go over a lot of trigggers.
With scripts you can "generalise" and make it work for a lot more usecases.
There's a lot of scripts out there already, but sometimes being able to write our own is the difference between a mission you could like and the mission you love to make!

This Course

Like mentioned earlier the goal of this course is to learn you how to code in Lua.
And to an extension to learn you how to utilise it in DCS as it can be an incredibly powerful tool.
I'll do this by guiding you through all the features Lua has to offer.
I built this website to make it easily accessible, free and also to make it possible to add a lot of nice ways to help you learn.
Videos will be added as they are created. I am a better programmer than I am a video creator. Patience is key.
What I did add though is way to write, highlight and test your code.
A huge shoutout to onecompiler that makes this possible. Read more about them and their goals on their website!
You will however not get any intellisense (which helps a lot in an IDE.), but sometimes it's good to learn without it!
As to how it works practically:
First you can go over the assignment and try to understand what is asked.
You will write code and you'll see the highlights as you'd normally see in your IDE. These help a lot when writing your code.
Once you think you've captured all the requirements you can hit "test" and all test cases are ran.
You'll get real-time feedback on your code and if it passed all the tests or not.
